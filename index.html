A pipeline, not a prompt

From raw codebase
to executable plan

Seven stages. One directional flow. Verification at every step.

Phase I
Recon
Map the repo: stack, conventions, build commands. Ingests ADRs, PRDs, DESIGN.md — so decided tradeoffs stay decided.
Phase II
Audit
Nine parallel subagents — correctness, security, performance, test coverage, tech debt, dependencies, DX, docs, direction.
Phase III
Vet
Advisor re-reads every cited location. False positives dropped. Wrong attributions corrected. Rejections recorded.
Phase IV
Prioritize
Findings ordered by leverage — impact divided by effort, weighted by confidence. You choose what becomes plans.
Phase V
Plan
One file per finding. Self-contained specs with code excerpts, exact steps, verified commands.
Phase VI
Execute
Cheaper model in an isolated worktree. Re-runs every done criterion, checks scope, reads diff against intent.
Phase VII
Reconcile
Next session: verify what landed, refresh what drifted, unblock what got stuck. Living documents.
Plans that execute

Not essays. Specifications.

Three properties that make every plan executable by the weakest model. No shared context, no improvisation.

S
Self-contained
All context inlined — exact paths, code excerpts, repo conventions, verified commands. No "as discussed above."
V
Verification gates
Every step ends with a command and expected output. Done criteria are machine-checkable. No subjectivity.
B
Hard boundaries
Explicit out-of-scope lists and STOP conditions — so small models never improvise past their competence.
You → /improve → plans/ → done

Intelligence compounds where it matters — understanding the codebase, judging what's worth doing, writing the spec. The skill never implements. The plan is the product.
