A pipeline, not a prompt

From raw codebase to executable plan

Seven stages. One directional flow. Verification at every step.

I
Recon
Map the repo: stack, conventions, build commands. Ingests ADRs, PRDs, DESIGN.md.
II
Audit
Nine parallel subagents — correctness, security, performance, test coverage, tech debt, DX, docs, direction.
III
Vet
Advisor re-reads every location. False positives dropped. Rejections recorded.
IV
Prioritize
Findings ordered by leverage — impact ÷ effort, weighted by confidence.
V
Plan
One file per finding. Self-contained specs with code excerpts and verified commands.
VI
Execute
Cheaper model in isolated worktree. Re-runs done criteria, checks scope.
VII
Reconcile
Verify what landed, refresh what drifted, unblock what got stuck.
Plans that execute

Not essays. Specifications.

Three properties that make every plan executable by the weakest model.

S
Self-contained
All context inlined — exact paths, code excerpts, repo conventions, verified commands.
V
Verification gates
Every step ends with a command and expected output. Machine-checkable done criteria.
B
Hard boundaries
Explicit STOP conditions — so small models never improvise past their competence.
You→/improve→plans/→done

Intelligence compounds where it matters — understanding the codebase, judging what's worth doing, writing the spec. The plan is the product.
