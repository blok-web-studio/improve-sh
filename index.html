How it works

Three steps,
one pipeline

improve doesn't just fix code. It audits, plans, and executes — turning expensive model runs into reusable blueprints.

01

Scan

Deep audit of your entire codebase. Analyzes architecture, dependencies, and surfaces hidden issues that context-window models miss.

02

Plan

A complete execution plan with file paths, diffs, dependency order, and rationale. Not a vague suggestion — a real blueprint.

03

Execute

Feed the plan to any model. Even lightweight ones produce production-grade changes when they follow a clear plan.

terminal
# generate an execution plan $ improve plan --model opus-4 ✓ 7.2k tokens · 12 steps · ready # execute with any downstream model $ improve run --model gpt-4o-mini ✓ 12/12 steps completed · 3 files changed
Why improve

Spend once on thinking,
execute anywhere

Premium models are expensive but brilliant. Cheap models are fast but limited. improve bridges the gap.

Model-agnostic plans

Your audit plan works with any downstream model. One Claude Opus run powers 50+ GPT-4o-mini executions. No vendor lock-in.

Cost-optimized

One $0.50 audit powers dozens of $0.01 runs. Your budget goes further when thinking is reusable.

Structured output

Every plan includes exact file paths, diff snippets, dependency order, and test instructions. No ambiguity, no guesswork.

Full codebase scale

Audits 15,000+ files per run. Analyzes architecture, dependency graphs, and cross-module impacts that single-file tools miss.
